# Title 
Boards Communicate Via Computer

## Status
Pending

## Context
We need the boards to communicate

## Decision
I'm proposing that the boards communicate through a companion program on the computer that the robot is relayed to

## Consequences
- Commands are sent directly to the board that needs them rather than from the comptuer, to one board and then the other
- head and body can become physically independent
- Save GPIO pins on both boards
